# SPA-003 — span names have bound cardinality

**Impact:** Important (weight 30) · **Target:** span · **Source:** spec ·
**Confidence:** sampled only

Distinct span names per service must stay bounded; raw URLs or IDs in span
names destroy latency breakdowns and span-metrics pipelines.

**Detection.** HLL sketch per service over span names
(`span_name_cardinality` aggregate, two-generation tumbling window). Fails at
`params.max_span_names` (default **200**).

**Threshold caveat.** The upstream spec leaves SPA-003's criteria TODO; 200
is Argus's initial pick, flagged for real-world tuning before v0.1
(DECISIONS.md).

**Remediation.** Template `unbounded-span-name`: name spans by route template
(`GET /users/{id}`), never raw path.
