# MET-005 — metric names do not embed the unit

**Impact:** Normal (weight 20) · **Target:** metric · **Source:** spec ·
**Confidence:** sampled only

`collection.duration.seconds` embeds what the unit field should say; when the
unit changes, the name lies.

**Detection.** Name matched against `params.unit_in_name_pattern` (time,
byte, and percent units, separated by `.`/`_`). Prometheus-convention
`_total` is a counter suffix, not a unit — deliberately not matched.

**Remediation.** Template `unit-in-metric-name`: drop the suffix, set the
unit field; note OpenTelemetry-to-Prometheus exporters re-append unit
suffixes on the Prometheus side automatically.
