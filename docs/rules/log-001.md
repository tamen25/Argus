# LOG-001 — debug logs are not enabled in production

**Impact:** Important (weight 30) · **Target:** log · **Source:** spec ·
**Confidence:** sampled only

DEBUG-band records (severity 5–8) MUST NOT be observed from services whose
`deployment.environment.name` matches `params.prod_env_pattern` (default
`^prod`). Debug volume in prod is pure cost.

**Fidelity note.** The spec's "longer than 14 days" clause needs history a
stream window cannot see; Argus fails on any DEBUG observed in prod during
the window (DECISIONS.md). Services without an environment attribute are not
judged by this rule — but they fail [ARG-RES-003](arg-res-003.md).

**Remediation.** Template `log-level-abuse`: set prod log level ≥ INFO, or
drop DEBUG in the collector.
