# SPA-002 — traces do not contain orphan spans

**Impact:** Normal (weight 20) · **Target:** span · **Source:** spec ·
**Confidence:** sampled only (span topology has no backend verifier)

Any span referencing a `parent_span_id` MUST have that parent present in the
same trace. Orphans = broken context propagation.

**Detection.** The trace tracker accumulates span/parent IDs per (service,
trace) on the sampled stream — bounded (LRU-capped traces, capped spans per
trace) — and judges only **completed** windows, so in-flight traces are never
scored. A service fails when its orphaned-trace ratio exceeds
`params.max_orphan_ratio` (default **0.1**).

**Sampling caveat (important).** Head sampling drops trace fragments
legitimately; the observed ratio is an upper bound. The 10% default absorbs
that; tune it to your sampler. Findings stay `confidence: sampled` forever —
tail-sampled or unsampled pipelines can lower the threshold toward 0.

**Remediation.** Template `broken-context-propagation`: check propagator
config (W3C tracecontext everywhere), instrument gateways, verify async hops.
