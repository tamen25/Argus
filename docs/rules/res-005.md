# RES-005 — service.name is present

**Impact:** Critical (weight 40) · **Target:** resource · **Source:** spec ·
**Confidence:** poller-verifiable (`mimir_service_presence`)

Resource attributes MUST contain a `service.name` key with a non-empty string
value. Telemetry without it lands in the `<unknown>` service bucket and cannot
be attributed, correlated, or costed.

**Detection.** Every item's resource attributes are checked on the sampled
OTLP stream. The Mimir poller additionally marks services that appear with a
proper `job` label as verified-passing (absence in Mimir is *not* treated as
failure — a service may legitimately emit no metrics).

**Remediation.** Set `service.name` in the SDK (`OTEL_SERVICE_NAME`) or inject
it at the collector (template `missing-service-name`, ships later in Phase 1).

See also [ARG-RES-001](arg-res-001.md) for SDK-default names like
`unknown_service:java`, which pass this rule's letter but not its spirit.
