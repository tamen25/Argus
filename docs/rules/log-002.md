# LOG-002 — log records have severity set

**Impact:** Important (weight 30) · **Target:** log · **Source:** spec ·
**Confidence:** sampled only

Records MUST carry a severity. Argus first normalizes SeverityText-only
records (TRACE/DEBUG/INFO/WARN/WARNING/ERROR/FATAL, case-insensitive) to the
OTel number scale at conversion — this rule fails only when neither a number
nor a recognizable text is present. Zero tolerance (any unset record fails
the service).

**Remediation.** Template `log-severity-unset`: fix the log-bridge mapping
(most common: custom appenders dropping level on the OTLP path).
